# 프로젝트 개발 원칙

## 개발 방식
이 프로젝트의 모든 개발 작업은 `/dev-team` 스킬을 기본 베이스로 진행합니다.

새로운 기능 구현, 버그 수정, 화면 개선 등 **모든 코딩 작업 요청 시**:
1. `/dev-team [요구사항]` 형식으로 실행
2. 6단계 전문가 팀(기획→UI/UX→개발→보안→QA→최종테스터)이 순서대로 협업
3. 최종 테스터의 APPROVED가 날 때까지 피드백 루프 반복

## 팀 구성
| Phase | 역할 | 책임 |
|-------|------|------|
| 1 | 🗂️ 시니어 기획자 | 요구사항 분석 및 기능 명세 |
| 2 | 🎨 시니어 UI/UX 엔지니어 | 화면 설계 및 사용자 흐름 |
| 3 | 💻 시니어 개발자 | 코드 구현 |
| 4 | 🔒 Secure 코딩 개발자 | 보안 취약점 점검 및 수정 |
| 5 | 🧪 QA 테스터 엔지니어 | 테스트 케이스 실행 및 결함 보고 |
| 6 | ✅ 최종 테스터 | 요구사항 충족 여부 최종 판정 |

## 사용 예시
```
/dev-team 거래처별 송장 자동 생성 기능 추가. PDF 출력 및 이메일 발송 포함.
```

## 예외
- 단순 질문, 코드 설명, 파일 읽기 등 **구현을 수반하지 않는 작업**은 dev-team 없이 직접 처리
